# Version 3 — Compiled, Cloud-Built, and on npm

Version 3 of the VisualText / NLP++ engine is a major milestone: analyzers can now be
**compiled** to native C++, **built in the cloud** for every platform, and the engine is
**available on npm**. This page is the user-facing summary; for the full story see the
blog post linked at the bottom.

[← Back to Help home](../home.md)

## 🛠 One-click compiling

You can turn an analyzer's NLP++ rules into a **native C++ library** without any C++
knowledge — straight from the extension.

- Faster execution and a deployable artifact you can ship without exposing source.
- The generated code includes type-correct overloads, provenance comments for tracing,
  and correctness fixes (null-pointer guards, evaluation-order fixes).
- Three flavors — **Compile Analyzer and KB**, **Compile Analyzer Only**, and **Compile KB**.

See **[Compiling Analyzers](../compiling.md)** for how to do it in the extension.

## ☁️ Cloud-built for every platform

A new **cloud compile service** handles cross-platform native linking, producing binaries
for **Windows, Linux, and macOS** on hosted runners. It's integrated into the VS Code
extension with progress shown in the status bar — so you can compile even without a local
C++ toolchain.

## ⚡ Lazy-loading dictionaries

The engine now **loads dictionary words on demand** instead of reading entire lexicons up
front. The result is **faster startup and a smaller memory footprint**, which matters most
for large knowledge bases. New supporting functions include `loadkbb` and `loaddict`.

## 📦 Now on npm (and better on PyPI)

- **[Node.js](https://www.npmjs.com/package/nlpplus) (new):** `npm install nlpplus` — a native addon that bundles the runtime.
- **[Python](https://pypi.org/project/NLPPlus/) (enhanced):** `pip install NLPPlus` — adds `compile()` and `cloud_compile()` APIs and context-manager support.

## More improvements

- **Zone-based processing** in the shared package analyzers — built for real-world HTML
  and Markdown rather than line-by-line text.
- International support in the Telephone analyzer.
- New `_xVAR("attribute")` match-list special; digits allowed in underscore-prefixed token
  names.
